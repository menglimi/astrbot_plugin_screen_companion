# 屏幕伙伴更新日志

## 2.9.5 (2026-04-06)

### 新增

- 新增更明确的内置识屏技能调用面，统一自然语言识屏的提示词拼装、隐式触发许可与执行入口，后续继续接 AstrBot 正式工具注册会更顺。
- 新增窗口标题应用描述映射模块，用于统一提取应用名、标题、场景和活动描述，帮助识屏回复更自然、更稳定。

### 优化

- 留空 `system_prompt` 或 `companion_prompt` 时，会更稳定地继承 AstrBot 当前人格，并把历史占位文案视为未填写。
- 自然语言识屏在群聊中的隐式触发更谨慎，通常需要更像是在明确叫到 Bot，减少普通聊天误触。
- WebUI 与配置说明同步改成“内置识屏技能”表述，配置入口更容易理解。

### 修复

- 修复陪伴模式说明容易让人误以为等于“开始自动观察”的文案问题。
- 修复自然语言识屏中管理员权限覆盖不完整的问题，现在会先过权限校验，再进入触发判断。
- 修复网页 / 报错类求助容易被应用启动器冲突规则误拦截的问题。
- 修复留空人格配置时，未必能正确沿用当前 AstrBot 人格的问题。
- 修复 WebUI 在插件重载时没有走到正确生命周期清理、以及同端口重启仍按“先起后停”导致旧端口残留占用的问题。

## 2.9.4 (2026-04-06)

### 修复

- 修复历史配置中异常 Windows 盘符路径格式可能导致插件加载失败的问题，例如 `:E:\...`、`/E:/...` 这类脏值现在会自动纠正后再使用。
- 清理发布包内不应携带的本地缓存与运行数据目录，降低跨机器安装时被旧环境残留影响的概率。

## 2.9.3 (2026-04-02)

### 新增

- WebUI 配置中心新增“基础设置 / 高级设置”切换，并提供游戏陪伴、工作陪伴、低打扰、轻量记录四个常用场景模板，减少首次配置时需要手动理解的大量参数。

### 优化

- 为窗口自动陪伴补上可配置的“窗口重连宽限期”，默认延长到 300 秒，更适合游戏结算、重开和重新匹配这类会短暂销毁窗口的场景。
- 窗口陪伴命中同一规则的新窗口时会优先续接当前会话，减少“结束提醒”和“开始提醒”在短时间内来回触发。
- 窗口陪伴会显式继承当前生效的自定义触发间隔和触发概率，WebUI 也会展示这组实际生效参数，避免和“窗口检查间隔”混淆。
- 设置页按功能开关动态隐藏子项，例如关闭窗口陪伴、录屏识屏、本地输入统计或日记后，不再默认展示对应的细分配置。

## 2.9.0 (2026-03-29)

### 新增

- 新增本地输入统计，可记录键盘、点击、滚轮和鼠标移动，用来补充更像 KeyStats 的轻量工作回顾。
- 新增工作脉搏、今日工作轨迹、应用 / 网站 / 页面级活动轨迹，以及更完整的活动统计页结构。
- 新增离开自动挂起、回来自动恢复和长时间离开轻量提醒，减少非观影场景下的无效打扰。
- 新增活动识别自定义规则，支持按行配置应用 / 站点别名，让轨迹更贴合个人工作流。
- 新增日记窗口轨迹补料，识屏素材不足时可自动从当天窗口 / 应用 / 站点轨迹中抽取代表性活动补进日记。
- 新增独立活动轨迹采集开关。开启后，即使没有启动自动观察，也会按固定间隔记录当前活动窗口，并尽量拆出应用 / 网站 / 页面轨迹。

### 优化

- 开始 / 结束文案使用 LLM 生成时，结束自动观察的回复会参考最近识屏结果与插件回复，让收尾更贴近当前场景。
- 活动回顾统一接入有效工作口径，启用本地输入统计后会尽量扣除长时间无键鼠输入的空闲。
- 浏览器窗口标题识别进一步细化，可更稳定地拆出应用、站点和页面标题。
- 活动页支持窗口标题脱敏，更适合长期挂着使用。
- WebUI 保存体验升级：配置中心和运行设置会在保存失败时优先区分“真正失败”和“WebUI 正在重连”，并在可能恢复后自动拉取最新状态。
- 活动页补充关键词 / 类型 / 来源筛选、统计说明和清空筛选入口，工作轨迹回顾更容易按场景聚焦。
- WebUI 服务自检补充详细检查项与建议卡片，能更直观看到静态资源、目录、输入统计、独立轨迹采集和窗口读取能力是否正常。
- 日记生成改为更稳的补写链路，待写素材会落盘保存，低观察量场景下也能生成简版日记。
- 将 `pynput` 并入基础依赖，默认安装即可使用本地输入统计，不再需要额外安装输入统计依赖文件。
- 独立轨迹采集会在自动观察或窗口自动陪伴真正运行时自动让位，继续优先使用识屏链路写入的活动轨迹，减少两套来源互相打架。

### 修复

- 修复 Linux / Docker 场景下 `pyaudio / portaudio.h` 导致插件整体安装失败的问题，并将麦克风监听依赖拆分为可选安装项。
- 修复 Windows 初装后 WebUI 运行状态接口的 `name 'shutil' is not defined` 异常，避免单个字段报错影响整页接口。
- 修复 WebUI session 清理日志乱码、旧活动时间戳展示异常、Safari 浏览器识别过宽等问题。
- 修复日记仅在精确分钟触发、插件重载后待写条目丢失、手动补写和切换目录状态不同步等问题。

## 2.7.2 (2026-03-20)

### 调整

- 更新插件版本号到 `2.7.2`。
- 新增敏感场景自动沉默：检测到支付页、密码管理器、登录/验证码相关界面、聊天窗口时，本轮只记录"检测到敏感界面，已跳过回复"，不再主动说话。
- 主动陪伴补充"沉默也是互动"规则，在变化不大或用户刚被打断等场景下，允许本轮只更新内部观察而不发言。
- 开始/结束文案改为会参考最近上下文，不再只走固定模板。
- `/kp`、`/kpr`、`/kps` 指令权限调整为管理员权限，不再允许所有人使用。
- 麦克风监听支持运行时动态启停：打开 `enable_mic_monitor` 无需重启插件即可生效，音量采样采用更稳定的实现。

### 优化

- 新增情绪记忆短缓存，能在"刚被打断 / 刚卡关 / 刚完成"后的几轮回复里延续前一段情绪，不再每次都像重新开场。
- 回复生成补充开场轮换约束，降低"我看了下""你现在像是在""可以试试"这类固定句式的连续复用。
- 主动陪伴链路进一步增强连续性和低打扰表现，让回复更像在旁边跟着当前节奏，而不是每轮都重新起势。
- 麦克风音量采样优化：优先查找默认输入设备，读取多段样本，避免 int16 平方溢出导致音量异常。

## 2.7.1 (2026-03-19)

### 调整

- 日记补强兜底逻辑，避免只剩标题和天气；日记页也不再重复渲染“今日概览”。
- 保留的观察记录从最多9条增加到最多50条，以覆盖更多场景。
- 休息提醒改为更严格的“推测作息 + 每天仅一次”，并弱化深夜时间提示词对连续催睡的诱导。
- 主动回复补充更强的连续性约束，减少重复称呼、重复开场和连续几条都重新起势的问题，现在bot的回复应该更加拟人自然了。

### 优化

- 主动陪伴会结合与上一条主动回复的时间间隔，区分“顺着接话”“自然续上”和“隔了一阵子后重新靠近”的语气。
- 建议型回复继续保持主导，但允许偶尔只做轻观察、共鸣或陪伴，降低机械感。
- Gemini 原生视频链路会尽量保留完整录屏以保住音轨，并明确要求模型只在能辨识音频时再参考音频。
- 录屏模式补充当前截图锚点，减轻“像在回复上一轮画面”的延后感。
- 自动化后台任务在录屏模式下改为每次触发现录一整段，再结合当前截图分析，更贴近触发当下。
