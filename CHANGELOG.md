# Changelog

All notable changes to this project will be documented in this file.

## [1.8.0] - 2026-03-03

### Added
- 新增用户休息时间设置（rest_time_range），避免bot过早地开始催用户休息
- 为配置项添加了条件显示功能，当主要功能关闭时隐藏对应的子配置

### Fixed
- 修复了配置 schema 中的 'type' 错误，确保插件能够正常加载

### Changed
- 隐藏了机器人的人格设定，简化配置面板
- 将 KPI 互动强度的描述改为“快捷切换配置”并移到了配置面板底部
- 隐藏了 KPI 互动强度的详细配置，只保留了级别选择
- 隐藏了空日记相关配置和日记被偷看警告

## [1.7.0] - 2026-03-03

### Added
- 新增日记自动撤回配置（diary_auto_recall）
- 新增日记自动撤回时间配置（diary_recall_time）
- 新增日记被偷看回应提示词配置（diary_response_prompt）
- 新增`/kpi diary [YYYY-MM-DD]`命令，用于查看特定日期的日记
- 新增`/kpi recent-diaries [天数]`命令，用于查看近几天的日记

### Fixed
- 修复了kps命令停止任务后仍在执行的问题，现在任务会被正确取消和清理
- 修复了麦克风依赖无法正确识别的问题，在requirements.txt中添加了pyaudio和numpy依赖

### Changed
- 移除了日记自动发送功能，改为通过kpi子命令拉取
- 在不活跃时间段直接停止任务，不再进行不必要的判定和token消耗
- 在调用视觉API之前检查活跃时间段，节省token
- 为所有任务添加了任务标识，使日志更加清晰
- 优化了互动模式的切换，修改互动模式后相应的参数一起更改，若用户修改参数，则自动切换成自定义
- 增强了麦克风依赖检查的诊断信息

## [1.6.0] - 2026-03-01

### Added
- 新增版本历史记录

### Changed
- **代码结构优化**：拆分了庞大的`_analyze_screen`方法，将其拆分为多个独立的功能方法，符合单一职责原则
- **数据路径规范**：修复了数据路径硬编码问题，使用`StarTools.get_data_dir()`替代手动拼接路径
- **导入规范优化**：将所有函数内部的标准库导入移到了文件顶部，符合PEP 8编码规范
- **硬编码字符串处理**：将诺星缘人设Prompt抽离到文件顶部的常量定义中，使代码更整洁
- **框架兼容性**：移除了废弃的@register装饰器，确保与AstrBot框架的兼容性
- **异常处理优化**：改进了天气API调用的异常捕获粒度，确保非核心功能的失败不会影响核心功能的执行

## [1.5.0] - 2026-02-28

### Added
- 新增环境感知功能（节假日识别、天气感知、系统状态检测）
- 新增日记功能，支持生成风格化的日记
- 新增自定义监控任务功能
- 新增麦克风监听功能
- 新增用户行为学习功能

### Changed
- 优化了消息发送逻辑，支持分段发送
- 改进了场景识别能力
- 增强了配置文件的可读性和可维护性

## [1.0.0] - 2026-02-27

### Added
- 初始版本发布
- 基本的屏幕截图和分析功能
- 自动监控和手动触发功能
- 个性化人格设定
- 多场景互动模式
