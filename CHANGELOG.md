# 屏幕伴侣插件更新日志

## 2.6.1 (2026-03-16)

### 新增

- 新增 `/kpi ffmpeg [路径]` 指令，可自动将 ffmpeg 复制到插件目录的 bin 文件夹，方便用户快速配置录屏功能。

### 调整

- 优化观察记录去重逻辑：只有当"窗口"和"场景"同时相同时才进行相似度比较，保留更多有价值的观察记录。
- `recording_fps` 配置项支持小数（如 0.5 代表 2 秒/帧）。

### 修复

- 修复 `_conf_schema.json` 中 `recording_fps` 类型定义错误（`number` 改为 `float`）。

## 2.6.0 (2026-03-15)

### 新增

- 新增截图 / 录屏识别素材开关，支持在图片识别和视频识别之间切换。
- 新增录屏识别链路：本地通过 `ffmpeg` 录制桌面 mp4，并在识别触发时提取最近一段视频参与分析。
- 新增 `/kpr` 指令，用于立即执行一次录屏识别；当全局仍处于截图模式时，也会先临时录制一段桌面再继续处理。
- 新增 Gemini 原生多模态直连优化：当当前聊天 provider 可识别为官方 Gemini API 时，图片优先走 `inline_data`，视频优先走 `Files API`。
- 新增 `allow_unsafe_video_direct_fallback` 开关，用于控制在非 Gemini 原生链路下是否允许继续兼容视频直发。
- 新增 `ffmpeg_path`、`recording_fps` 和 `recording_duration_seconds` 配置项，支持自定义 `ffmpeg` 路径、录屏帧率和录制时长。
- WebUI 运行状态页新增“最新素材预览”能力，可直接查看最近保存的截图或录屏。

### 调整

- `use_external_vision` 现在明确支持两条识别链路：外部视觉 API 两步识别，以及直接发送给 AstrBot 的多模态识别。
- `/kp` 与 `/kpr` 的命令语义固定化：`/kp` 始终执行截图识别，`/kpr` 始终执行录屏识别，不再受全局识别素材开关影响。
- 默认录屏参数调整为 `1 fps`、`10 秒`，降低录屏素材体积、上传负担和 token 风险。
- 录屏模式下查找 `ffmpeg` 的顺序调整为：插件目录 `bin/ffmpeg.exe`、配置项 `ffmpeg_path`、系统 `PATH`。
- 非 Gemini 原生链路上的视频直发默认会被安全拦截，避免将整段视频 base64 直接塞入兼容消息结构导致请求体和 token 消耗过高。
- WebUI 增补识屏模式、识别链路和运行态说明，并为共享目录截图模式补充最新截图回退预览。

### 修复

- 补充统一的运行时状态兜底初始化，修复热更新或旧实例复用时缺少新字段导致的 `AttributeError`。
- 补齐自动观察、窗口陪伴、录屏识别、WebUI 启停和后台调度相关入口的运行时状态检查，降低运行中断概率。

## 2.5.3 (2026-03-13)

### 新增

- 新增“共同经历”长期记忆，记录用户明确提到的与 Bot 一起看过的影视、玩过的游戏、做过的测试和识屏互动。
- WebUI 新增表格化统计看板。
- 统计看板新增时间范围筛选，支持今天、近 7 天、近 30 天、全部时间和自定义起止日期。

### 优化

- 优化开始/结束任务的 LLM 回复，使其更稳定地应用人格语气，减少机械式系统播报。
- 开始/结束任务回复现在会优先使用当前会话的人格上下文。
- README 与 WebUI 版本信息同步更新到 `2.5.3`。

## 2.5.2 (2026-03-11)

### 新增

- 实现错误纠正和持续性记忆功能。
- 实现自我形象识别和记忆。
- 实现 `webui` 命令返回端口信息。

- 修复 `TCPSite.__init__() got an unexpected keyword argument 'sock'` 错误。
- 修复 `charset must not be in content_type argument` 错误。
- 修复 `json_response() got an unexpected keyword argument 'charset'` 错误。
- 修复 WebUI 中的中文乱码问题。

### 优化

- 优化 `_build_vision_prompt` 方法，按重要性排序组织提示词，以降低 LLM 响应时间。
- 更新默认端口从 `8898` 到 `6314`。
- 减少自动切换端口的次数从 `10` 到 `3`。
